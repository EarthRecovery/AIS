# Aistoria 幻想物语 —— 「模拟即写作」小说创作系统设计文档

> 目标：一个**通用**的叙事引擎——能建模任意类型小说，让 AI **自主演进、发展故事，直到达到结局**。
> 一句话：**世界(模拟)产出实质，故事(叙事智能)决定走向与终点，小说家(Writer)把它写成书。**

---

## 0. 总纲：三层架构（WORLD → STORY → MANUSCRIPT）

这是整个系统的顶层心智模型，后面所有 Phase 都挂在它下面。

```
┌─ MANUSCRIPT 稿件层 ── 读者看到的正文（小说家 Writer 渲染；可编辑/导出）
│
├─ STORY 故事层（叙事智能）★ 当前最大缺口
│    冲突 · 角色目标 · 弧光 · 线索/张力 · 戏剧结构 · 结局目标
│    + 自主故事导演(规划) + 结局判定 + 批评/编辑闭环
│
└─ WORLD 世界层（已基本具备）── 角色/关系/物品/地点/认知/记忆/事件 + 场景模拟
```

**核心判断**：现在系统建模了 WORLD（世界在运转）和 SCENE（场景对话），但没有 STORY。
所以它能"永远演下去"，却不知道"在讲什么故事、讲到哪、离结局差什么"，**不会收束到结局**。

**为什么这套是通用的**：冲突、目标、弧光、张力、结构、结局是**一切小说**共有的抽象；
类型(玄幻/谍战/言情/悬疑…)只改变*内容*，不改变这套*骨架*。在这一层建模 = 能建模"所有小说"。

### 0.1 STORY 层需要的系统（自主演进到结局的关键）

| # | 系统 | 作用 | 它解决"自主到结局"的哪一环 |
|---|---|---|---|
| S1 | **故事蓝图 + 结局目标** | 中心前提、中心冲突、意图结局(或可接受结局集)、主题 | 北极星：没有目标，"直到结局"就没方向 |
| S2 | **戏剧结构 + 张力曲线** | 弧线位置(铺垫→升级→高潮→收束)、目标张力形状 | 节奏：何时升温、何时喘息、何时进高潮 |
| S3 | **冲突与驱动（引擎）** | 中心+次级冲突；角色相互碰撞的目标/欲望 + 利害 | **自主性**：角色主动追目标→自然生事件，而非被动等指令 |
| S4 | **线索/张力追踪器** | 每条线(主线/支线/伏笔)的状态(开/升级/已收)与张力 | 结局条件之一：主要线索都收束 |
| S5 | **角色弧光追踪** | 每个主角色的"预期转变 + 进度" | 结局条件之一：弧光走完 |
| S6 | **自主故事导演 / 规划器** ★ | 评估故事状态→决定下一个戏剧动作(引入/升级/复杂化/反转/收束)→选 POV/定本幕戏剧功能→交模拟层演出→读后果→循环 | **把"会模拟的世界"变成"会发展到结局的故事"** |
| S7 | **结局判定器** | 判断中心冲突解决/弧光完成/主线收束→宣布完成或需尾声 | **终止条件**：没有它循环永不结束 |
| S8 | **批评 / 编辑闭环** | 一致性检查、停滞/重复检测("推进剧情了吗")、质量批评→重导向/重写 | **长程不崩**：自主跑久必然漂移，靠它兜底 |
| S9 | （支撑）设定 bible/术语/全局时间线/多 POV | 一致性底座 | 长篇不自相矛盾 |

**基石**：S3 冲突驱动 + S6 自主导演 + S7 结局判定 —— 前两者让它"自己发展"，后者让它"能结束"。

**现实提醒**：AI 自主生成连贯且有满意结局的长篇是前沿难题。可行路径=**渐进**：先搭骨架 + 强批评闭环，
质量随迭代提升。够得着的近期目标是"**自主把一个有明确冲突与结局目标的中短篇推到结局**"。

### 0.2 建议搭建顺序（总）

1. **Phase 1（成文闭环）**：能"选 POV → 生成一章正文 → 编辑/锁定 → 导出"。（见 §5）
2. **STORY 最小闭环**：S1 故事蓝图+结局目标 → S6 自主导演 → S7 结局判定。让它能**自己跑向结局**。
3. **STORY 增强**：S3 冲突 / S4 线索 / S5 弧光 / S2 结构。让它跑得**好**。
4. **质量兜底**：S8 批评/编辑闭环（长程一致与不停滞）。
5. **支撑与进阶**：S9 设定 bible/术语/时间线/多 POV；按题材加 §9 的 F1–F11。

# 第二部分 · 成文层（Writer / Manuscript）

> 本部分聚焦"把模拟内容写成小说正文"，是 §0.2 里的 **Phase 1（成文闭环）**。
> 一句话：**交流(模拟)产出实质，小说家(Writer)把它写成书。**

参照样本：《诡秘之主》开篇（第一部·小丑 / 第一章 绯红）。其文本特征是本设计的"目标态"基准：
第三人称限知视角、大量主角内心独白、层层递进的感官描写、通过"记忆碎片"自然带出世界设定、
克制的信息差（主角不知道自己怎么死的、枪从哪来）、时代质感（煤气灯、便士、瓦斯计费器）。

---

## 1. 核心判断

| | 现在产出 | 小说需要 |
|---|---|---|
| 形态 | 「【角色】台词」逐句对话记录（像剧本） | 叙事散文：描写 + 内心独白 + 动作 + 对话 |
| 视角 | 无统一视角，各角色平铺发言 | 明确 POV（如第三人称限知，跟随主角） |
| 信息 | 全量对话 | 只叙述 POV 角色所知（信息差是张力来源） |

**结论**：当前系统是「世界模拟引擎」，产出的是"发生了什么"(ground truth)。
**缺失的关键一环**：一个「小说家 agent」+「稿件层」，把模拟内容渲染成读者看到的正文。

### 1.1 两份文风样本的启示：小说家必须能切换"笔法"

参照样本其实给了**两种截然不同的注册(register)**，小说家 agent 必须都能写：

| | 样本 A（第 1–3 章 绯红） | 样本 B（第 133–136 章 怨魂之战） |
|---|---|---|
| 类型 | 慢节奏·发现/内心戏 | 快节奏·多人动作戏 |
| 构成 | 几乎全是 POV 内心独白 + 感官描写，对话极少 | 对话 + 动作 beats + POV 战术推演穿插 + 超凡能力描写 |
| 信息 | 通过"记忆碎片"自然带出世界设定 | 战前简短问答交代规则(净化/驱邪/序列)，战中靠动作推进 |
| 句式 | 长句铺陈、缓慢推进 | 短句、拟声("砰！轰隆！")、快速切镜 |

**对设计的影响**：
- 小说家不能用单一模板。要按**场景类型/节奏**自适应：独处发现戏 → 放大内心与描写；冲突动作戏 → 动作 beats + 对话 + POV 即时判断("这和莎伦预计的不一样……")交织，配合拟声与短句。
- POV 的**战术心理**是动作戏的灵魂（克莱恩边打边算计对方序列、能力、退路）——这要求模拟层把 POV 的"判断/意图"也作为 beat 产出（见 Phase 2）。
- 文风样例摄取(Phase 3)应能从**多段样本**提炼出"注册库"（慢戏/快戏/对话戏…），而非一个笼统风格。

---

## 2. 现状盘点（已具备，直接复用）

- **世界与角色**：World / Character（含 stats、status、location）/ Worldview（公共设定 + 仅作者可见背景秘密）。
- **认知与记忆**：AgentMemory（短期=最近场景 / 长期=重要、可沉淀进 self_summary）、Belief（主观认知，可与真相不符）、MentalState（情绪/目标/动机/自我摘要）。
  → 这正是小说的"人物 + 信息差"。例：克莱恩的"记忆碎片"= 给 POV 角色的记忆；他不知道的事 = 他没有的记忆。
- **关系/物品/地点/世界事件**：Relationship（有向）、Item、Location、WorldEvent（追加式时间线）。
- **结构与节奏**：剧本大纲 outline（章节 beats，可编辑）、beat_index（当前章）、章节制推进、细粒度快照与回退。
- **导演/裁判(Keeper)**：open_scene（开幕、选 POV 候选角色、衔接上一幕）、judge_round（每轮结算状态：记忆/位置/物品/数值/情绪/目标/动机/摘要/关系/认知）、consolidate_memory（记忆沉淀）、write_outline（编剧写大纲）。
- **感知(perception)**：每个角色只看到它该知道的（世界常识 + 自己短/长期记忆 + 当前场景知识），**绝不暴露 is_true**。
- **生成**：流式逐 token 输出（step / run-scene / run-chapter / outline 均有 stream）。
- **参考文风**：World.style_guide（已加），注入生成上下文。
- **前端沙盘**：章节→场景两层目录、人物菜单（可编辑细节）、可编辑大纲、控制按钮（下一轮/新建场景/完成本场景/完成本章/新建章节/回退）、流式直播。

---

## 3. 目标架构

```
设定世界观 + 背景秘密 → 建角色(人格/秘密/关系) → 写章节大纲
   → 每章：模拟跑出"发生了什么"(对话 + 动作 + 心理 beats + 状态变化)
   → 选 POV → 小说家渲染成正文 → 作者审阅/改/锁定
   → 编译导出整本小说
```

两层分离：
- **模拟层（已存在）**：世界真值。谁在哪、知道什么、说了/做了什么、状态怎么变。
- **稿件层（新增）**：面向读者的正文。由小说家 agent 从模拟层 + POV + 文风渲染而来，可人工编辑。

---

## 4. 数据模型新增

1. **World**
   - `pov_character_id`（可空）：默认 POV 角色。
   - `narration`：叙述视角设置（如 `third_limited` / `first` / `third_omniscient`）。
   - 已有 `style_guide`：参考文风（可由"文风样例"提炼）。

2. **ChapterManuscript（新表）**：一章一份正文。
   - `world_id`、`beat_index`（对应大纲第几章）、`title`
   - `pov_character_id`、`content`（正文 markdown/纯文本）
   - `status`：draft / locked（锁定后不被重写覆盖）
   - `word_count`、`updated_at`
   - 一章可由其下多个 scene 拼成；重写=重新渲染未锁定章节。

3. **（可选）Term / 术语表**：world_id + 词条（序列、途径、货币、神名…）→ 一致性约束。

4. **（可选）Volume / 部卷**：在章节之上加一层（第一部 小丑）。初期可用大纲分组或一个 `volume` 字段代替。

## 5. 分阶段计划

### Phase 1 — 小说家 + 稿件层（地基，最高优先）
- [ ] World 加 `pov_character_id` / `narration`；新增 `ChapterManuscript` 表 + 迁移。
- [ ] **Novelist agent**（core/agent）：
  - 输入：本章/本幕的 setting、在场角色及其相关状态、发生的对话/动作/心理 beats、POV 角色、`style_guide`、上一章结尾（衔接）、相关世界常识。
  - 关键：**叙述只基于 POV 的感知**（复用 perception）→ 信息差自然成立。
  - **按场景类型自适应注册**：慢戏(发现/内心)→放大内心+描写、长句；快戏(冲突/动作)→动作 beats+对话+POV 即时判断+拟声短句。
  - 输出：该章/幕的**正文散文**（描写+内心独白+动作+对话，比例随注册变化），第三人称限知或指定视角。
- [ ] Service：`render_chapter(world_id, beat_index, pov?)` → 拉该章所有 scene 的内容 → 调 novelist → 存 ChapterManuscript（流式可选）。
- [ ] 路由：`POST /world/{id}/manuscript/{beat}/render`、`GET /world/{id}/manuscript`、`PATCH /manuscript/{id}`（人工编辑/锁定）。
- [ ] 沙盘：「生成本章正文」按钮 + 正文阅读/编辑面板 + 锁定。
- [ ] **导出**：`GET /world/{id}/export?fmt=md|txt` → 按 部/章 拼全文下载。

### Phase 2 — 让"原材料"更适合写小说
- [ ] 每轮产出从"只有台词"扩成 **台词 + 动作 + (POV)内心想法/判断** beats。独处场景=纯内心戏(样本 A)；动作戏需 POV 的**战术意图/即时判断**(样本 B：克莱恩边打边算计序列/能力/退路)。
- [ ] 导演可插入 **旁白/环境 beats**（对房间、血月的层层描写）。
- [ ] Novelist 按注册调配比例：慢戏放大内心+感官；动作戏用动作 beats+对话+拟声短句+POV 判断交织。

### Phase 3 — 作者工作流 & 结构
- [ ] **部/卷/章** 三级结构。
- [ ] **改稿闭环**：正文可手改 → 锁定 → 只重写未锁定章节；按字数/节奏目标控制每章长度。
- [ ] **文风样例摄取**：粘**多段**范文（慢戏/快戏/对话戏各一）→ 提炼出"**注册库**"而非单一风格，按场景类型取用（比手写 style_guide 更准、更全）。

### Phase 4 — 一致性 & 打磨
- [ ] **术语表/设定 bible**：锁定专有名词，渲染时强约束。
- [ ] **一致性检查 agent**：扫正文标矛盾（知道了不该知道的、名词不统一、时间线错乱）。
- [ ] 角色口吻一致性。

## 6. 关键技术点

- **POV × 感知 = 信息差**：小说家以 POV 角色的 perception 为"可叙述边界"。主角不知道的，叙述就不写或只写"他猜测/困惑"。这是把已有的主观认知系统直接变现为文学价值的关键。
- **文风落地**：style_guide 注入小说家 system prompt；进阶用"文风样例 → 特征提炼"。
- **衔接与长度**：渲染本章时带上一章结尾若干段，保证过渡；按目标字数把 beats 展开成完整段落（样本每章约 2000–3000 字，叙述为主）。
- **真值不漂移**：正文必须忠于模拟层的状态与事件（谁在哪、拿了什么、知道什么），小说家只负责"怎么写得好看"，不负责"改剧情"。改剧情仍走模拟层/导演指示。
- **成本**：渲染是每章一次大调用（可流式）；模拟是多轮多调用。两者解耦，便于分别优化。

## 7. 里程碑（建议顺序）

1. **M1（地基）**：Phase 1 全做完 —— 能"选 POV → 生成本章正文 → 编辑/锁定 → 导出全文"。跑通一章《诡秘之主》式开篇正文即验收。
2. **M2（材料）**：Phase 2 —— 内心戏/旁白 beats，正文质量显著贴近样本。
3. **M3（工作流）**：Phase 3 —— 部/卷结构、改稿闭环、文风样例摄取。
4. **M4（一致性）**：Phase 4 —— 术语表 + 一致性检查，支撑长篇不崩。

## 9. 从全书提炼的可新增特性（书证）

读《诡秘之主》全本（490 万字 / 6 部 / 1394 章）后，按"在书中的中心地位（词频）"挑出最值得做成**系统一等公民**的特性。这些大多超出 Phase 1–4，属于"让长篇/复杂世界写得住"的进阶能力。

| # | 书中机制（词频佐证） | 对应系统特性 | 复用/新增 | 建议阶段 |
|---|---|---|---|---|
| F1 | **序列/途径**（序列 3857、途径 2017、失控 693） | **力量/进阶体系**：定义途径→序列(等级)→能力→进阶条件(演绎法)→失控风险；角色据此一致地行动与成长 | 新增 PowerSystem/Pathway 实体 | 进阶 |
| F2 | **多重身份/化名**（样本：道恩·唐泰斯"三层身份"；克莱恩=愚者=莫里亚蒂=世界…） | **角色化名/身份层**：一角色多身份，标记每个场景用哪个身份；跨身份保密 | 新增 Identity，挂 Character | 进阶 |
| F3 | **塔罗会**（678）+ 代号（倒吊人1373/正义1492/太阳1877…） | **加密社团/会面**：成员用代号、互不知真名、信息高度不对称 | 复用 perception/belief + 代号层 | 进阶 |
| F4 | **教会/神系/派系**（教会 1703、愚者 3057） | **组织/派系**：教会、社团、势力，含成员、立场、政治；驱动跨线冲突 | 新增 Organization/Faction | 进阶 |
| F5 | **占卜/预知/灵感**（占卜 2197、灵感 275） | **预知/伏笔机制**：占卜结果、"灵感"危险预警；与伏笔追踪联动 | 新增，配合 F6 | 进阶 |
| F6 | **超长伏笔与回收**（全书招牌） | **伏笔追踪器**：登记"埋点→回收"，一致性检查确保回收、不烂尾 | 新增 PlotThread/Foreshadow | Phase 4 强化 |
| F7 | **神奇物品/封印物/灵性**（非凡 3954、封印物 964、灵性 1120） | **道具体系强化**：物品带能力/副作用/灵性/封印态/来历，而非纯库存 | 扩 Item 模型 | Phase 2/进阶 |
| F8 | **失控/精神污染**（失控 693） | **理智/污染等异常数值规则**：阈值→后果（疯狂/反噬），影响行为与叙述 | 复用 stats + 规则层 | 进阶 |
| F9 | **多部/多线/多 POV + 全局时间线**（6 部各换主线/地点） | **多 POV 与并行线 + 世界时间线**：不同线并行推进、可切 POV、事件全局排序 | 扩 World/Manuscript | Phase 3 强化 |
| F10 | **设定一致性**（货币便士/镑/苏勒、地名、历法、血月） | **设定 bible / 术语表 / 地图 / 历法**：锁定专有名词与世界规则，渲染强约束 | Phase 4 术语表扩展 | Phase 4 |
| F11 | **章节短标题**（绯红/情况/活尸和狼人/毒素瓶） | **章节标题自动生成**：据本章正文起一个凝练标题 | 小说家附带 | Phase 1 附带 |

**落地建议**：Phase 1–4 仍是主干（先能"生成→编辑→导出一章正文"）。F1–F9 属于"复杂长篇支撑层"，等核心写作闭环跑通后，按题材需要逐个加——优先级最高的是 **F6 伏笔追踪** 与 **F10 设定 bible**（直接决定长篇会不会崩/烂尾），其次是 **F2 身份层** 与 **F3/F4 社团/派系**（谍战、群像、信息差题材的灵魂）。
